## Codebase Patterns
- Routes are Fastify plugins: `export async function xxxRoutes(server: FastifyInstance)` registered in `server/index.ts`
- DB uses drizzle-orm with better-sqlite3, imported from `../db/connection.js`
- Schema tables exported via barrel in `server/db/schema/index.ts`
- IDs use `crypto.randomUUID()`, not auto-increment
- JSON schema validation on Fastify request bodies (not Zod)
- Return 201 for creates, 204 for deletes, 404 for not found
- Default workspace pattern: `getOrCreateDefaultWorkspace()` returns the "Default" workspace ID
- Abilities schema already has: id, workspaceId, name, type (skill|plugin|mcp), active, config (JSON), version, createdAt
- Junction tables use composite primary keys via `primaryKey({ columns: [...] })` in drizzle
- Stage-ability routes live in pipelines.ts since they're nested under `/api/pipelines/...`
- Use `innerJoin` on junction tables to return related entity data in GET endpoints
- Frontend API hooks live in `src/api/*.ts`, follow pattern: export interface + useX query + useMutateX with invalidateQueries
- `apiFetch` defaults to `Content-Type: application/json`; override header for raw text endpoints
- Layout store (`src/stores/layout.ts`) holds `activeAbilityId` for passing selected ability to detail views
- ReactFlow: import `@xyflow/react` and its CSS (`@xyflow/react/dist/style.css`); container MUST have explicit height (use `h-full` with flex parent)
- Graph components live in `src/components/graph/`; `GraphView.tsx` is the thin wrapper in views/
- Custom ReactFlow nodes: wrap in `memo()`, define `nodeTypes` object outside component, register via `nodeTypes` prop on `<ReactFlow>`
- AgentNode data type: `{ label, nodeType, model?, systemPrompt?, status?, onOpen?, onDelete? }` — pass callbacks via node data
- Sidebar loading/error pattern: match the projects panel (Loader2 spinner, AlertCircle + retry button)
- `typeBadge` helper in LeftSidebar maps ability types to color classes (skill=violet, plugin=emerald, mcp=cyan)
- Custom ReactFlow edges: use `getBezierPath` + `BaseEdge`, define `edgeTypes` outside component, register via `edgeTypes` prop
- Edge data type `CustomEdgeData` has `condition` field: always | on-success | on-failure | on-decision
- `EdgeLabelRenderer` for floating UI elements (buttons) at edge midpoints; use `deleteElements` from `useReactFlow()` for programmatic edge removal
- ChainEditor uses `ReactFlowProvider` wrapper; hooks like `useReactFlow()` / `screenToFlowPosition` must be called inside an inner component
- AddNodeMenu tracks both screen coords (for CSS popup placement) and flow coords (for `node.position`) — two coordinate systems
- Node type definitions (color, label, description) are in `AddNodeMenu.tsx` as `nodeTypeEntries`
- Inject callback props (onOpen, onDelete) into nodes via a `useMemo` wrapper — keeps refs fresh without re-creating nodes
- `useNodesState<Node>([])` returns generic `Node`; cast to `Node<AgentNodeData>` when passing to typed components
- NodeInspector lives in `src/components/graph/NodeInspector.tsx`; opens via `inspectedNodeId` state in ChainEditor
- ChainToolbar is a pure presentational component — action callbacks are passed as props from ChainEditorInner
- Export JSON uses Blob + createObjectURL + anchor click pattern for file downloads
- Chain persistence: `chainDefinition` column on projects table is `text({ mode: "json" })` — drizzle handles JSON serialize/deserialize
- Chain API: GET/PUT `/api/projects/:id/chain` — returns `{ version, nodes[], edges[] }`, defaults to empty chain if null
- Auto-save pattern: `useRef` timer + `setTimeout(1000)` debounce; `hasLoadedRef` flag prevents overwriting user edits with stale query data
- Deserialize chain → ReactFlow: map saved nodes to `Node<AgentNodeData>` with `type: 'agent'`, map edges with `type: 'custom'`
