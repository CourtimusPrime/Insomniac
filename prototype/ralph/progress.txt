## Codebase Patterns
- Server code lives in `server/` and is typechecked with `tsconfig.server.json` (`npx tsc --project tsconfig.server.json --noEmit`)
- Routes are Fastify plugins exported as `async function xyzRoutes(server: FastifyInstance)`
- Routes are registered in `server/index.ts` via `await server.register(xyzRoutes)`
- Use `.js` extensions in imports (ESM module resolution)
- Pre-existing typecheck error in `server/marketplace/client.ts:142` — not introduced by us
- `getRunnerForProject(projectId)` is exported from `server/routes/localhost.ts` to access dev server state from other modules
- Fastify wildcard routes use `server.get<{ Params: { "*": string } }>("/path/*", ...)` — access via `request.params["*"]`

## 2026-03-22 - US-005
- Added `GET /api/browser/proxy/*` route to `server/routes/browser.ts`
- Proxy requires `?projectId=xxx` query parameter to identify which dev server to proxy to
- Uses `getRunnerForProject()` exported from `localhost.ts` to discover the dev server port
- Proxies requests via Node's native `http.request` to `http://127.0.0.1:{port}/{path}`
- Streams response back including content-type, content-length, and cache-control headers
- Returns 502 if dev server is not running or not reachable
- Adds CORS headers (Access-Control-Allow-Origin for localhost:5173)
- Files changed: `server/routes/browser.ts`, `server/routes/localhost.ts`
- **Learnings for future iterations:**
  - Fastify wildcard routes: use `{ Params: { "*": string } }` generic and `request.params["*"]` to access the captured path
  - Fastify querystring validation uses `schema.querystring` (not `schema.query`)
  - Node's `http.request` callback receives an `IncomingMessage` (readable stream) — Fastify's `reply.send()` accepts streams directly
  - Avoid unused parameters in Promise constructors — TypeScript's `noUnusedParameters` will catch them
  - The proxy streams responses (no buffering), making it suitable for HTML, CSS, JS, and assets
