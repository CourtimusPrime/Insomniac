## Codebase Patterns
- Theme JSON files go in `src/themes/data/` and use VS Code color token keys (e.g., `editor.background`, `sideBar.background`)
- The `.gitignore` has a `data/` rule — theme data needs `!src/themes/data/` exception
- Theme registry in `src/themes/registry.ts` — add import + entry to `THEMES` array
- `useTheme()` hook provides `themes` list from registry — ThemeSwitcher dropdown auto-renders all entries
- `mapVSCodeColors()` maps VS Code tokens to AppTheme tokens via `TOKEN_MAP` in `mapping.ts`
- Layout store (`src/stores/layout.ts`) uses Zustand — add new state + actions there, localStorage for persistence
- SettingsView tabs: add type to `SettingsTab` union, entry to `TABS` array, render with `{activeTab === 'x' && <Tab />}`
- Pinned themes stored in layout store as `pinnedThemes: string[]`, persisted to `insomniac-pinned-themes` in localStorage
- Server routes pattern: export async function `xRoutes(server: FastifyInstance)`, register in `server/index.ts`
- DB schema pattern: UUID primary keys via `crypto.randomUUID()`, `integer("x", { mode: "timestamp" })` with `sql\`(unixepoch())\`` for timestamps
- Use `real()` for decimal columns (e.g., estimatedCost) — SQLite has no native decimal type
- Frontend API hooks: `useQuery` with `refetchInterval` for polling, files in `src/api/`
- `process.cpuUsage()` returns microseconds — divide by elapsed time × cores for percentage
- Usage queries use `coalesce(sum(...), 0)` to handle empty tables — always returns numbers, never null
- SQLite `strftime('%Y-%m-%dT%H:00:00Z', col, 'unixepoch')` for hourly time-series bucketing
- CSV export: set `Content-Type: text/csv` and `Content-Disposition: attachment` headers on Fastify reply
- Usage API hooks: `useUsageSummary()` (5s poll), `useUsageTimeline()` and `useUsageBreakdown()` (10s poll) in `src/api/usage.ts`
- For CSV export downloads, use a plain `<a href="..." download>` — no need for fetch + blob when the server sets Content-Disposition
- VS Code JSON files (keybindings.json, settings.json) may contain comments — strip `//` and `/* */` before `JSON.parse()`
- File upload pattern: hidden `<input type="file">` + `useRef` + button with `onClick={() => ref.current?.click()}`
- Settings API stores individual keys via `PUT /api/settings/:key` — use `editor_` prefix for editor settings (e.g., `editor_fontSize`)
- Onboarding gate: `useSetting('onboarding_completed')` in App.tsx — render wizard when value is not `true`
- Full-screen overlay: `fixed inset-0 z-50` with `bg-bg-default` for modal flows that replace the entire shell
