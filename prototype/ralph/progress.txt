## Codebase Patterns
- Schema files use `sqliteTable` from `drizzle-orm/sqlite-core` with `text`, `integer` columns
- Primary keys use `text("id").primaryKey().$defaultFn(() => crypto.randomUUID())`
- Timestamps use `integer("column_name", { mode: "timestamp" }).notNull().default(sql\`(unixepoch())\`)`
- Foreign keys use `.references(() => parentTable.id)` with snake_case column names
- Enums use `text("column", { enum: [...] })` (SQLite text enum pattern)
- Booleans use `integer("column", { mode: "boolean" })`
- Barrel export in `schema/index.ts` uses `.js` extension in imports
- Migrations generated via `npx drizzle-kit generate`, applied via `npx drizzle-kit migrate`
- Typecheck: `npx tsc --noEmit -p tsconfig.server.json` for server, `npx tsc --noEmit` for client
- JSON columns use `text("col", { mode: "json" }).$type<T>()` for typed JSON in SQLite
- Encryption utility at `server/crypto/keys.ts` — uses AES-256-GCM with scrypt-derived key
- Ciphertext format: `iv:authTag:ciphertext` (hex-encoded, colon-separated)
- Encryption key from `INSOMNIAC_SECRET` env var or auto-generated `data/.secret`
