## Codebase Patterns
- Routes are Fastify plugins: `export async function xxxRoutes(server: FastifyInstance)` registered in `server/index.ts`
- DB uses drizzle-orm with better-sqlite3, imported from `../db/connection.js`
- Schema tables exported via barrel in `server/db/schema/index.ts`
- IDs use `crypto.randomUUID()`, not auto-increment
- JSON schema validation on Fastify request bodies (not Zod)
- Return 201 for creates, 204 for deletes, 404 for not found
- Default workspace pattern: `getOrCreateDefaultWorkspace()` returns the "Default" workspace ID
- Abilities schema already has: id, workspaceId, name, type (skill|plugin|mcp), active, config (JSON), version, createdAt
- Junction tables use composite primary keys via `primaryKey({ columns: [...] })` in drizzle
- Stage-ability routes live in pipelines.ts since they're nested under `/api/pipelines/...`
- Use `innerJoin` on junction tables to return related entity data in GET endpoints
