## Codebase Patterns
- Schema files use `sqliteTable` from `drizzle-orm/sqlite-core` with `text`, `integer` columns
- Primary keys use `text("id").primaryKey().$defaultFn(() => crypto.randomUUID())`
- Timestamps use `integer("column_name", { mode: "timestamp" }).notNull().default(sql\`(unixepoch())\`)`
- Foreign keys use `.references(() => parentTable.id)` with snake_case column names
- Enums use `text("column", { enum: [...] })` (SQLite text enum pattern)
- Booleans use `integer("column", { mode: "boolean" })`
- Barrel export in `schema/index.ts` uses `.js` extension in imports
- Migrations generated via `npx drizzle-kit generate`, applied via `npx drizzle-kit migrate`
- Typecheck: `npx tsc --noEmit -p tsconfig.server.json` for server, `npx tsc --noEmit` for client
- JSON columns use `text("col", { mode: "json" }).$type<T>()` for typed JSON in SQLite
- Encryption utility at `server/crypto/keys.ts` — uses AES-256-GCM with scrypt-derived key
- Ciphertext format: `iv:authTag:ciphertext` (hex-encoded, colon-separated)
- Encryption key from `INSOMNIAC_SECRET` env var or auto-generated `data/.secret`
- Service classes use `ProviderRegistry` pattern — class with CRUD methods, lives in `server/providers/`
- Public types strip sensitive fields (e.g. `apiKeyEncrypted`) and add derived flags (e.g. `hasApiKey`)
- Static model definitions live in `server/providers/models.ts` — `PROVIDER_MODELS` constant + `getModelsForProvider()` function
- Ollama and OpenRouter have empty static lists — Ollama models fetched dynamically, OpenRouter accepts arbitrary strings
- `OllamaProvider` class in `server/providers/ollama.ts` — async `fetchModels()` and `ping()`, takes optional baseUrl
- Fastify route plugins: `async function routeName(server: FastifyInstance)`, registered in `server/index.ts` via `await server.register(fn)`
- Upsert pattern: check existing row with `.get()`, then `insert` or `update` — no native upsert in better-sqlite3 drizzle
- Fastify route order: static paths (e.g., `/ollama/models`) must be registered before parametric (e.g., `/:id/models`)
- Route validation: JSON Schema in `schema.body` with `additionalProperties: false`
- Frontend API hooks live in `src/api/` — TanStack Query with `useQuery` for reads, `useMutation` for writes, invalidating on success
- Frontend types mirror server public types (no sensitive fields like apiKeyEncrypted)
- Adding a new view: (1) add to `MainView` type in `src/stores/layout.ts`, (2) create component in `src/components/views/`, (3) add conditional render in `src/components/shell/MainView.tsx`
- Settings view is accessed via toolbar icon, not via the header tab bar
- Frontend forms needing `workspaceId`: get it from `useProjects()` first project (always seeded)
- Form visibility pattern: `useState(false)` toggle + `onClose` callback prop to form component
- Reusable UI components live in `src/components/ui/` — use `useQueries` for parallel dynamic fetches
- Project preferences (default model) fetched via `useProjectPreferences(projectId)` — returns `null` if none set
- Pipeline stage creation: `useAddStage(pipelineId)` mutation posts to `/api/pipelines/:id/stages`
