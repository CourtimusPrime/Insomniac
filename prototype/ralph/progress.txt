## Codebase Patterns
- Server code typechecked with `npx tsc --noEmit -p tsconfig.server.json`
- Server routes are Fastify plugins registered in `server/index.ts`
- Database uses Drizzle ORM with better-sqlite3; schemas in `server/db/schema/`
- Project module uses ESNext with bundler module resolution
- Integration services go in `server/integrations/`
- Cloned projects live under `data/projects/` directory
- Route handlers share `workspaceId` via closure from plugin load
- LocalhostRunner is per-project; route layer uses a Map<projectId, LocalhostRunner> in server/routes/localhost.ts
- WebSocket log streaming uses `broadcast("devserver:log", { projectId, line })` from ws/handler.ts
- Client API hooks go in `src/api/<resource>.ts` using TanStack React Query (`useQuery`, `useMutation`)
- `apiFetch` from `src/api/client.ts` is the fetch wrapper for all API calls
- Active project tracked via `useProjectsStore((s) => s.activeProjectId)` from `src/stores/projects.ts`
- WebSocket event handling in `src/hooks/useWebSocket.ts` — add cases to the `handleEvent` switch to invalidate query keys
- Settings stored in `settings` table as key-value pairs (key: text, value: JSON) scoped to workspaceId
- Settings API: GET `/api/settings/:key` and PUT `/api/settings/:key` with `{ value, category? }` body
- Settings hooks in `src/api/settings.ts`: `useSetting(key)`, `useSaveSetting()`, `useTestSlackWebhook()`
