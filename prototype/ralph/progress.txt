## Codebase Patterns
- Schema files use `sqliteTable` from drizzle-orm/sqlite-core with `.js` extension in imports
- Primary keys use `text("id").$defaultFn(() => crypto.randomUUID())`
- Timestamps use `integer("created_at", { mode: "timestamp" }).default(sql\`(unixepoch())\`)`
- Booleans use `integer("col", { mode: "boolean" })`
- JSON columns use `text("col", { mode: "json" })`
- Barrel exports in schema/index.ts use `.js` extensions
- Migration workflow: `npx drizzle-kit generate` then `npx drizzle-kit migrate`
- Typecheck: `npx tsc --noEmit`
- Fastify routes are registered as plugins in server/index.ts
- Seed pattern: define data array + `seedIfEmpty()` function that checks `.all().length > 0`, called at route plugin registration time
- Route plugins call seed functions at top of the exported function (e.g., `await seedBuiltInTemplates()`)
- Apply template pattern: copy `chainDefinition` from template to project, increment `installCount`
- Marketplace client uses a mock store index (array of MarketplaceItem) — will be replaced by GitHub fetch later
- Marketplace routes use `getOrCreateDefaultWorkspace()` helper (duplicated — consider extracting to shared util)
- Pagination pattern: parse page/limit from querystring, slice results, return `{ items, total, page, limit, totalPages }`
- MarketplaceClient class pattern: constructor takes optional storeUrl, async methods return typed data
- `getOrCreateDefaultWorkspace()` helper returns the default workspace ID (duplicated across routes — consider extracting)
- ChainDefinition format: `{ version: 1, nodes: [{id, type, label, model, systemPrompt, status, abilities, position}], edges: [{id, source, target, condition}] }`
- Node types: trigger, prototyper, builder, tester, reviewer, auditor
- Edge conditions: always, on-success, on-failure, on-decision
